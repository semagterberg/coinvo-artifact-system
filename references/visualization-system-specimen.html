Pattern library · named layouts so far
Structure as a system
Constant rail plus proportional bar plus parallel stream. Use when the topic is a composition with parts that do not compete.
Forcing functions
Evidence rows with one measured number per lever. Use when each decision needs an external force to defend it.
Phase curve
Bars across phase columns showing evolution. Use to show a number was earned, not declared.
Outcome map
One-to-one pairs across two columns. Use when the topic maps cleanly onto a parallel framework.
Tier ladder
Ranked bars with a native metric per row. Use to rank entries when each row has its own ruler.
Trap curve
Two lines crossing with an annotated callout. Use when the topic is an inverse relationship people mistake for a leak.
EV bars
Same effort, different yield. Use when a small percentage beats a big percentage in absolute terms.
Health gauges
Per-band reference scales. Use when comparing tiers needs a different ruler per tier.
Two-state contrast
A versus B side-by-side. Use when the right answer becomes obvious only against the wrong one.
Stat cards
Big number with one-line context. Use as accents inside other patterns or as a 3-up summary row.
Pattern library is open-ended. New angles get added when built. The list above is prior art, not a constraint.
